++ Serving Correctly Sized Images ++
// Optimizing Images //

Serve Image at a dedicated size for the element you're putting it in.
Ex: The component below is hardcoded to be 1140px wide.
The image will take up 50% of that - (1col of our 2col grid)
As such, the largest our image will be, is 570px wide
Which means there is no sence serving a 4k image.

What about Mobile?
Mobile resolution is less than 570px; So size is fine - NOT!
To cater to Mobile's pixel dense screens - we need to double the size!
Serve 1 image@ 570px wide - image
Serve 1 image@ 1140px wide - image@2x

One step further - Make the Pngs into Webps!
Img 1.png (363 kb) >> Img 1.webp (42kb)
Img 1@x2.png (1138kb) >> Img 1@2x.webp (94kb)
Wowee, thats a mega reduction!!!
