All about Fonts + Sizes.

**Take the "Mobile First" design approach**
Inspecitng Moshify example, There are 2 fonts with varying weights and sizes.
Lets select our own fonts from Google Fonts

Using Google's CND we'll link up the "prompt" font style.
With pasting... in our head.
<link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> <link href="https://fonts.googleapis.com/css2?family=Prompt:ital,wght@0,400;0,800;1,200&display=swap" rel="stylesheet">

Then creating the CSS rule:
font-family: 'Prompt', sans-serif;

Standardization

Then Set universal sizing... 1rem = 10px
Convert default size from 16px to 10px by

html {
/* Default is 16px, *62.5 = Set font size down to 10px. But maintain scalability! */
font-size: 62.5%;
}

Scalable Sizes

Then inspect our design sheet, Identify spacing in Px, then convert to rem
Then set rems for each heading.

h3 {
/* 10px x 3 = 30px, but its scalable! */
font-size: 3rem;
margin-bottom: 1rem;
}

Margins!

Our margins aren't behaving as expected... why?
Because of margin-collapsing
the bottom-marin of our h3 is 1rem, but our top-margin of the p below
is 25px (greater than 1rem). As such the collapsed (or shared) margin is overwritten by default.
So lets set the margin-top of our p to 0.
2rem + 0... No margin-collapsing!

p {
margin-top: 0;
}

We need to set the bottom-margin for all our h1, h2, and h3 element.
Adhering to Object Oriented Programming and DRY principles,
(1 Does 3) Add one line of code, to one rule targeting three elements
(3 Each do 1) Rather than adding 1 line of code to three different rules targeting single elements

h1,
h2,
h3 {
color: var(--color-heading);
margin-bottom: 1rem;
}

Serving appropreate Sizes!

After Selecting the colors, standardizing our sizes, and setting our margins...
We're ready to consider what size media we're serving to.
Using... @media screen and (min-width: 1024px) {xxx}
We can check if the media screen is above or below 1024px and act accordingly...
Desktop vs Mobile

Desktop we've set h1, h2, h3, p To 7rem 4rem, 3rem and 2.4rem respectively.
Mobile we've set h1, h2, h3, p To 8rem, 4rem, 2.4rem and 1.8rem respectively.

These slight differences ensure the sence of scale is maintained throughout devices.
