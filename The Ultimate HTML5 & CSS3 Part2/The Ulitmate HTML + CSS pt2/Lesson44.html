Animations

Animations are a little like JavaScript Methods.
They need to be created, then called.
Animations are created with the @keyframes function.
Animations need a name - Whatever you like
Then animations need a set of instructions
0% {do X} ...... 100% {do X}

Then after its been created, we can call the @keyframe with two properties
animation-name: and animation-duration:
(We need both of these for the animation to function)
We can add additional rules with
animation-delay: Xs ~ To delay the animation by however many seconds
animation-iteration-count: X~ To set amount of loops
animation-timing-function: Keyword ~ to set timing style of animation
animation-direction: Keyword ~ to set animation to play forward, or reverse etc

Re-usability

Once we have created an animation, we can set up a dedicated class for it.
for instance "animation-NAME" is a good practice.
Now instead of the animation being hard-coded into a particular element,
it exists all by its self.

.box {
width: 100px;
height: 100px;
background: gold;
font-size: 1rem;
}

.animation-bloom {
animation-name: bloom;
animation-duration: 6s;
animation-timing-function: linear;
}


vs

.box {
width: 100px;
