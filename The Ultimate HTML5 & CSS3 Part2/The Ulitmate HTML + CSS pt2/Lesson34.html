Input Controls: Text Feilds

Input Attributes

the input element takes various attributes
value="" ~ Can be used to pre-populate an input field.
placeholder="" ~ Can be used to pre-populate an input feild with shadow-text that dissapears
readonly ~ User is not allowed to change the value within. (Value sent to server)
disabled ~ User is unable to even select the input field. (Value not sent)
maxlength="" ~ Sets a maximum character length - Used for validation
autofocus ~ On pageload, element with this attribute automatically gets focused

Capture Text;

to capture text, ensure that input is type="text"
This will ensure that a standard keyboard is shown on mobile too

Capture More than default;

To capture more input that the default allows, use the <textarea> element.
The resize is enabled by default, we can override that with CSS
resize: none;

This element will allow us to set name="" id="" cols="" rows=""
cols="" and rows="" take a character value,
as such 30 cols and 10 rows = 300 characters.

Capture Number;

To capture numbers, ensure that input is type="number"
This will not accept any keystrokes from A-Z characters, only numbers!
This will ensure that a number pad is shown on mobile too

Capture Password;

To capture passwords, ensure that input is type="password"
This ensures that instead of showing characters, black dots are shown.

Capture Date;

To capture dates, ensure that input is type="date"
This ensures that a calendar function is attatched to the input field.

Capture Email;

To capture dates, ensure that input is type="email"
This ensures that an email function is attatched to the input field.
This also performs basic validation, so the user has to type a 'valid' email address.
