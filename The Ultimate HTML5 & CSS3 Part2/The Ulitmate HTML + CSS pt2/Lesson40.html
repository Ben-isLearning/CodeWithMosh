Data validation + Submitting

Data Validation

Before submitting any form, its critcal to validate the data.
Validation ensures that everything behaves as desired.
For example, If someone creates an account with an email address as "123123",
The website will never be able to send an email to that.

Validation covers;
Required Fields, Emails, Numbers and Dates.

Hackers will supply "invalid" values to see what breaks, then see what breaks to gain access.

HTML5 elements come with a basic level of validation, however
anything truly meaningful is handled by JavaScript (and is out of the scope of this course)

The "required" attribute (Boolean Value) will set an input field to required.

The 'minlength="x" ' attrubute will set
the minimum character requirement of the input field.
The 'maxlength="x" ' attribute will set
the maximum character allowance of the input field.
Input will cease after given value.
the input type="email" attribute comes with its own validation.
Which requires an @ within the field.
the input type="date" attribute comes with its own validation.
Which requires a valid date.
the input type="number" attribute uses two different attributes
Rather than min-length="" and max-length=""
its simply min="" and max=""

Submitting the Form

Submit buttons can be created in two ways;
1) <button type="submit">Submit </button>
(Accepts Icon)
or
2) <input type="submit" value="Submit" </input>
(Only accepts Text)

Requirements

the <form> element requires two attributes to fully function.
1) action=""
Which defines where the form goes to be operated upon.
Which accepts an absolute or relative url
That is acted upon by server-side technology
(Node.js / Django / ASP.NET / Laravel / Ruby on Rails / Etc)

Formspree have a pretty good service Here to supply a form API

2) method=""
Which defines how we are sending data.
method="post" ~ Sends data to server ~ data stored in HTTP Request
method="get" ~ Used to request data from server ~ data is stored in URL
