Object-oriented CSS

A set of principals for creating reusable components

1) Seperate container from content
Setting a rule for .parent .child {xxx} is bad practice.
Because there is probably only one situation you can use it.
IRL example - rather than having creating a RED CHAIR rule
Create a RED rule and a CHAIR chair rule to serve the purpose.
You can then re-use the RED or the CHAIR rule elsewhere.
Set two rules, .parent {xxx} and .child {xxx}

2) Seperate Structure from Skin
Setting a rule;
.blue-box {height: 200px; width:200px; background-color: dodgerblue;}
that inludes both structural (size, padding etc) and skin (colors)
Is bad practice, because this cannot be reused.
Set two rules;
.blue {background-color: dodgeblue;}
.box {height:200px; width:200px;}
