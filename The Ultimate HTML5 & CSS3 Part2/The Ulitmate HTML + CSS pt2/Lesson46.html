CSS Best Practices

Follow a Naming Convention - Consistently!


1) Kebab Case - Seperate Multiple words with hypthens.
like-this-you-see

2) Camel Case - Starts with a lower case, but each word after is capitalized.
camelCaseLooksLikeThis

3) Pascal Case - Starts with capitalization, continues.
PascalCaseLooksLikeThis

4) Underscore Case - seperate words with _
looks_like_this_takes_ages_to_type

Create Logical Sections in Stylesheet

Even for large projects, split Style Sheets into multiples and access them via SASS

1) Basic Styles
Setting incredibly broad rules... *{} | html{} | body{} | box-sizing:

2) Typography
Setting styles for h1 - h6

3) Forms
Setting styles for input fields

4) Navigation Bar
Setting styles for Nav bars

Cluster sections together with appropreate notes.
Ideally we will be able to copy and paste
Line A - B of HTML and line X - Y of CSS and have it behave as expected.

Avoid Overspecific Selectors

div.nav > ul.items > li {} (Specificity 23)
Is an incredibly specific selector, which is incredibly hard to maintain.
And will certainly break with changes.

*Avoid child selectors where we can*
*Avoid type of elements*
*Create meaningful classes*
By removing the structural and relationship aspects, and creating a meaningful class,
The above code can become;

.nav-item{} (Specificity 10)

Avoid !important

If we need to use !important to make something work as intended,
We've broken something elsewhere by making it too specific.
