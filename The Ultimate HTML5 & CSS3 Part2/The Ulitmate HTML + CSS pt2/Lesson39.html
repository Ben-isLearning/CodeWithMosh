Grouping Fields / Hiding Fields

Grouping Related Fields

When forms get a little more advanced, we need to group related fields
into logical containers.
This is acheived by the <fieldset> element.
Which requires one <legend> element, and multiple <input> elements.

Or this can be acheived by the <section> element.
Which requires one <h2> element, and multiple <input> elements.

Hiding Fields

When the handing Forms, the front-end transfers data to the back-end (Server)
But how does the server know what has been changed?
Each input has a unique identifier ~ the value="" attribute.
Hidden values are used to identify files.

for example,
Red Sheep (value="123") goes out from the server...
Blue Crocodile (value="123") comes back...
While the file has undergone drastic changes,
it is still the same file as far as the server is concerned (value-"123").

Beware: Although these values are hidden to the user; they can still be inspected
using the Dev Tools.
Never store sesitive data in hidden fields.
